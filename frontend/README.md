# ChronoCanvasAI вЂ” Frontend

Next.js 14 frontend for AI-powered historical portrait generation and a RAG chatbot. Provides an elegant, animated UI for describing historical figures, selecting art styles, viewing generated portraits, and exploring the figures catalog вЂ” all with a floating AI assistant available on every page.

## Getting Started

### Prerequisites

- Node.js 18+
- The backend server running at `http://localhost:3001` (see [backend README](../backend/README.md))

### Setup

```bash
cd frontend

npm install

cp .env.local.example .env.local
# Edit .env.local if your backend runs on a different port

npm run dev
```

App runs at `http://localhost:3000`.

## Pages

### `/` вЂ” Home

Landing page with a hero section, live stats (portraits created, unique figures, styles), and quick-start templates. Clicking a template prefills the composer and navigates to `/generate`.

### `/generate` вЂ” Portrait Generator

The main generation interface. Users:
1. Type a description of a historical figure and scene
2. Select an art style (Renaissance, Anime, Sketch, Watercolor, Baroque)
3. Toggle AI enhance on/off
4. Click Generate вЂ” the portrait appears with a Framer Motion slide-up animation

Accepts a `?figure=<slug>` query parameter to pre-fill the composer from the figures catalog.

### `/gallery` вЂ” Community Gallery

Paginated, browseable grid of all portraits generated by users. Supports filtering by art style.

### `/figures` вЂ” Historical Figures

Searchable, filterable catalog of historical figures with era, origin, tags, and year-range filters. Each card links directly to the generator with a prefilled prompt. Selecting a card opens a side panel with full figure details.

## Key Components

### `ChatWidget`

`components/chat/ChatWidget.tsx` вЂ” Floating chat panel, globally mounted in `layout.tsx` so it's available on every page.

- Generates a stable `session_id` (UUID) on mount
- Sends messages to `POST /api/chat` and reads the SSE token stream
- Renders assistant responses via `react-markdown` (bold, lists, paragraphs)
- Shows a gold blinking cursor while a response is streaming
- **Reset button** (в†є) in the header: calls `POST /api/chat/{session_id}/reset`, swaps to the returned `new_session_id`, and clears the message list вЂ” all without deleting DB records
- Disabled state on reset button when no messages exist or a stream is in progress

### `PortraitComposer`

`components/home/PortraitComposer.tsx` вЂ” Core generation UI. Handles textarea input, style selection, `POST /api/generate`, loading shimmer, and Framer Motion image reveal.

### `FiguresGrid`

`components/figures/FiguresGrid.tsx` вЂ” Full-featured figures browser with live filter dropdowns populated from `/api/figures/meta`, client-side search, and pagination.

### `PortraitPageClient`

`components/home/PortraitPageClient.tsx` вЂ” Thin client bridge that receives a selected template from `QuickStartTemplates` and passes it as initial state into `PortraitComposer`. Keeps template selection logic out of server components.

## Customization

**Add art styles** вЂ” Edit the `STYLES` array in `lib/constants.ts`.

**Add quick-start templates** вЂ” Edit `QUICK_START_TEMPLATES` in `lib/constants.ts`.

**Add historical figures** вЂ” Use `POST /api/figures` or run the backend seed script. The figures catalog and chatbot FAISS index update automatically.

**Tune chatbot rules** вЂ” The system prompt lives in `backend/services/chatbot.py` (`_SYSTEM_TEMPLATE`). No frontend changes needed.

**Change the color theme** вЂ” Edit `tailwind.config.ts`. Custom palette: `cream`, `dark`, `gold`, `parchment`.

**Change fonts** вЂ” Edit the `next/font/google` imports in `app/layout.tsx`.

**Point to a different backend** вЂ” Update `NEXT_PUBLIC_BACKEND_URL` in `.env.local`.
